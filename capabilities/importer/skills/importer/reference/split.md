# Stage 4 — Split generic from personal

For every instruction, name, and value: *would a stranger installing this want exactly
this?*

- **Mechanism** → skeleton: flows, formats, schedule existence, safety rules, zone
  structure.
- **Nuance** → draft MOD.md: names, goals, phrasing preferences, specific hours, channel
  ids, tuned thresholds. Where nuance filled a structural slot, leave `{{mod: <slot>}}`
  in the skeleton.
- **Questions** → `ONBOARDING.md`: every extracted nuance implies a question for the next
  user. Draft the typed question list + a minimal script.
- Personal data that is neither (history, logs, old entries) → leave on the harness;
  GAP-note that it wasn't imported.

Unsure → nuance side + GAP note.
