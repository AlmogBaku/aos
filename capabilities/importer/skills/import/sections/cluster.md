# Stage 2 — Cluster

Group the inventory into candidate use cases: a cron whose prompt names a skill, the
skill, the persona fragment that sets its voice, and the KB zone it writes are one
cluster. Signals: shared names/prefixes, prompt cross-references, shared KB paths,
shared secrets, delivery targets.

The user named a use case ("my trainer") — everything else is scenery. Present the
cluster you believe implements it, plus a short "adjacent, excluded" list (artifacts
that touch it but seem to belong elsewhere), and let the user redraw the boundary before
you invest in mapping. Ambiguous membership is a question, not a guess.
